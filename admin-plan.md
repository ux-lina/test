# LINE Creators Market 어드민 시스템 작업 계획서

> 기반 문서: "LINE Creators 이달의 크리에이터 프로그램 전체 기획안"
> 작성일: 2026-04-20

---

## 1. 프로젝트 개요

### 목적
LINE Creators Market의 크리에이터 발굴 → 선정 → 계약 → 매니지먼트 → 정산으로 이어지는 **백엔드 파이프라인을 운영하는 어드민 시스템** 구축

### 범위
- 일본 6월 런칭 기준 MVP 우선 구축
- 웹 어드민
- 추후 대만·태국 확장을 고려한 국가별 분기 구조 설계

---

## 2. 문제 정의 및 해결 방향

| 문제 | 해결 방향 |
|------|-----------|
| 크리에이터 수는 방대하나 소비자가 누구에게 주목할지 모름 | 데이터 기반 월간 6명 선정 + 에디토리얼 노출 |
| 큐레이션 레이어 없음 (추천 노출이 전부) | 복합 지표 대시보드 + 선정 사유 코멘트 관리 |
| 주목할 만한 크리에이터 50~60명 수준, 롱테일 소진 | 공모전 병행으로 신규 크리에이터 유입 장치 마련 |
| 크리에이터 계약·정산 파이프라인 부재 | 독점(A)/비독점(B) 계약 자동화 + 정산 데이터화 |
| 선정 기준 투명성 부족 | 복합 지표 공개 + 선정 사유 대외 공개 원칙 |

## 3. 어드민 시스템 구성 모듈

### 3.1 크리에이터 대시보드 (발굴)
**목적**: 전체 크리에이터 중 월간 선정 후보 추출

**기능 목록**
- 전체 크리에이터 목록 (정렬·필터·검색)
- 복합 지표 스코어링 테이블
  - 매출 / 송신수 / 스탬프당 송신수 / SNS 지표
  - 가중치 슬라이더로 실시간 순위 재계산
- 상위 60~100명 자동 추출 뷰
- AI 생성 여부 플래그 (수동 입력)
- 국가별 필터 (일본 / 대만 / 태국)
- 정성 메모 입력란 (예: "SNS 반응이 매출 대비 뛰어남")

**데이터 소스**
- LINE Creators OASIS 연동 또는 CSV 업로드
- SNS 지표: 수동 입력 (OASIS 미지원 항목)

---

### 3.2 월간 선정 관리 (선정)
**목적**: 매달 6명 이달의 크리에이터 선정 및 회차 관리

**기능 목록**
- 회차 생성 (연도-월, 국가)
- 후보 리스트 → 최종 6명 선정 (드래그&드롭 또는 체크박스)
- 선정 사유 코멘트 작성 (크리에이터별)
- 선정 상태 관리: `후보` / `선정` / `비선정` / `보류`
- 대표 스탬프 1종 지정 (1인 1종 원칙)
- 회차별 이력 조회 (지난 회차 아카이브)
- AI 크리에이터 특집 / 국가별 특집 / 테마별 특집 회차 유형 설정

---

### 3.3 크리에이터 계약 관리 (계약)
**목적**: 선정 크리에이터 대상 A/B/C 옵션 계약 프로세스 자동화

**기능 목록**
- 이메일 발송 관리
  - A안(독점) / B안(비독점) / C안(불참) 옵션 포함 템플릿
  - 발송 이력 및 상태 추적 (`미발송` / `발송완료` / `회신대기` / `회신완료`)
- 크리에이터 응답 수집
  - A / B / C 선택 결과 기록
  - 응답 기한 알림 (D-day 표시)
- 계약서 자동 생성
  - 독점 계약서 템플릿 (A안)
  - 비독점 계약서 템플릿 (B안)
  - Google Docs 또는 PDF 출력
- 서명 상태 관리: `미서명` / `서명완료`
- A안 비중 모니터링 (A vs B 비율 대시보드)

**설계 원칙**
- 상위 작가의 독점 거부 흡수 → B안 경로 보장
- C안(불참) 선택 시 조용히 비선정 처리, 대외 노출 없음

---

### 3.4 콘텐츠 수집 및 관리 (관리)
**목적**: 크리에이터 인터뷰·스토리·SNS 콘텐츠 수집 및 에디토리얼 정리

**기능 목록**
- 크리에이터별 제공 데이터 수집 현황
  - 인터뷰 답변 / 제작 배경 스토리 / 외부 채널 콘텐츠 / 프로필 이미지
  - 수집 상태: `미요청` / `요청완료` / `수집완료` / `검수완료`
- 인터뷰 질문지 템플릿 관리
- 에디토리얼 편집 영역 (수집 데이터 → 최종 공개 콘텐츠 변환)
- 콘텐츠 아카이브 (회차별 누적 저장)
- 크리에이터 사이트 퍼블리싱 상태 연동

---

### 3.5 노출·채널 운영 (노출)
**목적**: 멀티채널 입체 노출 운영 현황 관리

**기능 목록**
- 채널별 노출 스케줄 관리
  - 스탬프샵 배너 / LINE 앱 푸시 / 스탬프 상세 페이지
  - X(트위터) / 인스타그램 공식 계정
- 배너 소재 업로드 및 버전 관리
- 푸시 발송 이력 조회
- LINE 공식 계정 팔로워 수 추이 (수동 입력 또는 API)
- 외부 공유 최소화 원칙 준수 체크리스트

---

### 3.6 투표·구매 연동 (투표)
**목적**: 구매 = 투표 통합 구조 운영

**기능 목록**
- 회차별 구매(투표) 집계 현황
- 가챠 리워드 발급 현황
  - 꽝 / 프렌즈 스탬프 / LINE FRIENDS 스토어 쿠폰 / 매장 쿠폰
  - 확률 설정 및 재고 관리
- 스탬프 결제 여부 확인 API 연동 상태 (선물 발송 판단용)
- 좋아요·공유 수 집계 (API 연동 또는 자체 구축)

---

### 3.7 공모전 관리 (공모전)
**목적**: 분기별 연 4회 공모전 운영 관리

**기능 목록**
- 공모전 회차 생성 (분기, 기간, 국가)
- 응모 크리에이터 목록 및 심사 현황
- 상금 지급 관리 (천만 원 수준)
- 수상자 → 이달의 크리에이터 후보 자동 편입
- 계약 조건 사전 정형화 템플릿 관리
- 런칭 타이밍 체크 (6월 런칭과 1회차 동시 시작 필수)

---

### 3.8 연말 어워드 관리 (어워드)
**목적**: 10개월 누적 선정자 50~60명 대상 연말 시상식 운영

**기능 목록**
- 연간 후보 풀 자동 집계 (월간 선정자 누적)
- 종합 지표 산출 (누적 구매 + 사용 이력 + SNS + 팬 반응)
- 수상 카테고리 및 수상자 관리
- 애뉴얼 리포트 데이터 출력 (보도자료화용)

---

### 3.9 파트너사 연결 (정산)
**목적**: 검증된 크리에이터 풀을 파트너사에 소개 → 계약 중개 → 정산

**기능 목록**
- 파트너사 목록 관리 (편의점, F&B, 리테일 등)
- 크리에이터-파트너사 매칭 제안 이력
- 중개 계약 진행 상태: `제안` / `협의중` / `계약완료` / `진행중` / `완료`
- 정산 산출 및 송금 데이터 기록
- A안(독점) 크리에이터: 직접 사업화 우선
- B안(비독점) 크리에이터: 파트너 매칭 중심

---

### 3.10 결과 리포트 (분석)
**목적**: 회차별 성과 데이터 로깅 및 다음 회차 크리에이터 자극

**기능 목록**
- 회차별 결과 리포트 페이지
  - 좋아요 수 / 조회 증가 그래프 / 구매 전환 데이터
- 크리에이터별 매출 변화 비교 (선정 전/후)
- A안 비중 모니터링 차트
- 국가별 성과 비교 (일본 / 대만 / 태국)
